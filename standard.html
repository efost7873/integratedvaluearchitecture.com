Integrated Value Architecture Standard

A public standard for how organizations structure decisions, track value, and stop the same work from getting stuck in the same places over and over again.

Read IVA Standard v1.1 (PDF)

Version 1.1 · February 2026

The IVA Standard exists because organizations keep running real work through structures that were never designed to carry it. Decisions pile up in the same places, approvals move slower than the work requires, and the same people keep getting pulled in to fix things that should have been handled earlier.

That usually gets treated like a management problem. It’s not. It’s a structure problem. The standard defines how to fix that by changing where decisions sit, how value is recognized, and how work moves through the organization.

What this looks like in real work

If you’re trying to figure out what category this fits into, it probably doesn’t. This is not a tool or a management layer. It changes how decisions and work are handled inside the organization.

Before

Work sits waiting for approval from someone who doesn’t have time to look at it. A manager gets pulled into decisions they shouldn’t own. Teams send work back and forth because nobody can make the final call. The same problems keep showing up and getting “fixed” again.

After

The decision moves to the person who actually owns it. Work gets finished without climbing up the chain first. Fewer things get kicked back or reworked. The same issue stops coming back because the structure that caused it changed.

What the standard actually does

This is not a reporting framework or a documentation exercise. It defines how the organization operates.

Separates value into real domains

Financial performance does not carry the full weight of the organization. The standard establishes distinct domains for operations, capacity, external impact, and learning so each one has standing instead of being forced through finance.

Defines who actually makes decisions

Decision rights are not left vague or assumed. The standard makes it clear who owns what so work does not keep climbing uphill or bouncing between teams.

Requires real evidence, not opinions

Entries are based on verifiable, durable evidence. If something changes, there is a record of what changed and why instead of relying on interpretation after the fact.

Prevents work from collapsing into one bottleneck

When everything has to pass through the same office, team, or role, the system slows down and people start working around it. The standard distributes authority so the organization can actually move.

What it changes in practice

This shows up in how work actually gets done, not in how it’s described.

Fewer decisions sit waiting on the same person to approve everything.

Teams stop sending work back and forth because nobody owns the final call.

Problems stop reappearing every few weeks under a slightly different name.

Leaders spend less time fixing work that broke somewhere else.

Staff can finish work without routing everything through management.

Capacity improves without immediately defaulting to hiring more people.

What this is not

If you’re trying to place this into something familiar, this is where people usually get it wrong.

This is not software.

This is not a dashboard or reporting tool.

This is not project management.

This is not workflow automation.

This is not a strategic plan.

This is not a one-time assessment that leaves the same structure in place.

The standard defines how the organization is structured so these problems stop happening in the first place. It is meant to be used, not just read.

Using the standard

Reading the standard and implementing it are two different things.

Public standard

The document above defines the structure, rules, and concepts that make the system work. It is publicly available so organizations can understand the approach.

Implementation work

Applying the standard requires mapping your actual decision flow, identifying where work is getting stuck, and redesigning how authority and responsibility are handled. That work happens through services.
